Ideas for Games

All of the games below would have multiplayer using ENet-CSharp or a C++ wrapper of ENet if the game ever transitions from Unity to OpenGL C++.

Idea #1

Star Citizen but its top down view and you start off as a primitive human working your way up the tech tree. I started this idea but quad tree chunking and pathfinding flowcharts on a deformed icosahedron were my roadblocks. Source Code / Pics of Progress

Idea #2

RimWorld but with infinite terrain, everyone joins in on one dedicated server. Think Rust but top down 3D view RTS.

Idea #3

Top down 3D view. Zombies. Infinite Procedural Terrain. Survive with friends. Build your fort. Progress up the tech tree. Think They are Billions but more zoomed in. Think Project Zomboid but 3D perspective view instead of isometric.

Idea #4

Kittens Game but reimagined and maybe with some visual graphics this time. Not to mention multiplayer elements.

Idea #5

Idea #6

Age of Empires 2 except there is no "match making". Everything is hosted on one dedicated server. There is one big infinite procedurally generated world. You log in, you join the world with everyone else. If you're not careful you could fight someone in the industrial ages while you are still in the primitive ages.
